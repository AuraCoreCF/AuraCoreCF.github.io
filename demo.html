Desktop Demo

Download, install, and run AuraCoreCF locally.

This page is the distribution and setup point for the AuraCoreCF desktop demo. The demo is designed to let users run the system locally, configure a model provider, and evaluate the application in a controlled environment without exposing internal implementation details or development-only materials.

AuraCoreCF is a local-first desktop application. It can run with a local model through Ollama or with a cloud model through a supported API provider, depending on how model settings are configured after installation.

What the demo is

The desktop demo provides a direct way to evaluate AuraCoreCF as an application rather than as a static description. It exposes the runtime surface users need in order to install the system, configure model access, and test behavior in a local environment.

The demo supports both local and cloud inference workflows so users can evaluate the application against the model environment that best matches their setup.

What the user controls

Users control model selection, provider configuration, API credentials, and local model connection settings from inside the application. This keeps deployment flexible while preserving a consistent application experience.

The active provider and model are shown inside the application interface so the runtime context remains visible during testing.

Model setup

Configure the active model provider

Open Preferences, go to AI Model, configure the inference source and model settings you want to use, then save and test the active provider before beginning evaluation.

Step 1. Open Preferences.
Step 2. Go to AI Model.
Step 3. Configure provider and model settings.
Step 4. Click Save Model Settings.
Step 5. Click Test Active Provider.
Inference modes
Auto (Recommended). Uses cloud if both API key and model are configured; otherwise falls back to local Ollama.
Local Ollama. Uses the configured local Ollama instance only.
Cloud API. Uses a configured cloud provider only.

Local Ollama setup

  • Set Inference source to Local Ollama, or leave it on Auto with no API key configured.
  • Enter the local model name you want to use in the Model field.
  • Confirm the local Ollama URL. The default is http://127.0.0.1:11434.
  • Click Refresh Ollama Models.
  • Click Save Model Settings.

Cloud provider setup

  • Set Inference source to Cloud API, or use Auto.
  • Select the cloud provider.
  • Enter the exact provider model ID in the Model field.
  • Paste your key into API key.
  • Confirm the correct API base URL.
  • Click Save Model Settings and then Test Active Provider.
Provider selection logic

How Aura decides what provider to use

Provider selection follows a fixed execution rule. If Inference source is set to Cloud API, Aura uses cloud only. If it is set to Local Ollama, Aura uses local only. If it is set to Auto, Aura uses cloud when both an API key and model are present; otherwise it uses local Ollama.

Local modeOllama only
Auto modeCloud if configured, else local
Use and support

Logs, troubleshooting, and support workflow

Export logs

Export error logs for support

If the demo encounters an issue, open Preferences, go to Privacy, then use Export Logs under the error log export area. Save the JSON file and include it with the support report.

Aura also keeps rolling runtime logs locally at AppData/Roaming/AuraCoreCF/logs/aura-errors.jsonl in installed builds.

Quick troubleshooting

Common issues

  • Cannot reach Ollama. Start Ollama and verify the configured local URL is reachable.
  • Cloud test failed (401/403). The API key is invalid or missing permissions.
  • Cloud test failed (404). The API base URL is incorrect for that provider.
  • Model not found. Verify exact spelling in the Model field.
  • Wrong provider still active. Re-open AI Model, confirm Inference source, save settings again, and re-test.
